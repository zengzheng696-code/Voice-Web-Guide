# Voice Web Guide

Voice Web Guide 是一个 Chrome 插件 MVP，用语音帮助用户理解和操作当前网页。它会读取当前页面的结构化信息，听取用户的问题，用同一种语言回答，并高亮相关按钮、字段、链接或页面区域。

它**不会**替用户点击、输入、提交、删除、购买或执行任何网页操作。它是一个实时网页指导助手，不是自动化 Agent。

## 为什么做这个产品

很多网页任务失败，并不是因为用户不会用浏览器，而是因为页面陌生、语言不熟、术语太多，或者操作风险不清楚。

典型情况包括：

- 中文用户进入全英文开发者控制台，想申请 API key，但不知道点哪里。
- 非技术用户看到 `Callback URL`、`Client Secret`、`Webhook endpoint` 等字段，不知道含义。
- 用户填写 SaaS 注册或配置表单时，不确定哪些字段必填、应该怎么填。
- 用户看到 `Revoke token`、`Delete project`、`Billing` 等按钮，想先确认会发生什么。
- 用户阅读英文价格、权限、账单、设置页面，只想要一句简短解释。
- 用户想边看网页边问：“这个按钮是干什么的？”“这里要填什么？”“我应该点这个吗？”

Voice Web Guide 就是为这些时刻设计的。

## 核心使用场景

### 1. 开发者平台指导

适用于 API 控制台、云服务后台、OAuth 配置页、Webhook 页面、SDK 文档等场景。

用户可以问：

- “这个页面怎么创建 API key？”
- “Callback URL 是什么意思？”
- “这个 API key 是不是要保密？”
- “Webhook 在哪里配置？”

插件会解释当前页面，并尽量高亮对应的按钮、字段或入口。

### 2. 跨语言网页辅助

当页面是英文，但用户习惯用中文提问时，用户可以直接用中文问，插件会用中文回答。

当用户用英文提问时，插件会用英文回答。

适合这些页面：

- 英文 SaaS 后台
- 开发者文档
- 账单和价格页面
- 开发者账号设置
- 国际产品的新手引导流程

### 3. 表单字段解释

插件会读取页面上可见的 label、placeholder、附近说明文字和鼠标位置，帮助解释表单字段。

用户可以问：

- “这里应该填什么？”
- “这个字段必须填吗？”
- “这个选项是什么意思？”
- “这里可以不填吗？”

插件不会自动填写，只会解释和建议。

### 4. 鼠标指向式提问

插件会在本地记录最近的鼠标位置。如果用户把鼠标移到某个按钮、输入框、错误提示、价格说明或权限标签上，然后问“这个是什么意思？”，模型会收到鼠标附近的页面上下文。

这适合用户不知道如何描述页面元素的情况。

### 5. 选中文本解释

用户可以用鼠标拖动选中网页上的一段文字，然后直接提问：

- “这段是什么意思？”
- “帮我解释一下这段英文。”
- “这段是在提醒什么风险？”

插件会把当前选中的文本和附近上下文传给模型，方便模型围绕选中文本回答。

### 6. 风险敏感操作提醒

模型会特别关注这些敏感概念：

- Billing
- Payment
- Delete
- Revoke
- Production
- Secret
- Token
- API key
- 权限变更

遇到这些场景时，插件应该提醒用户谨慎确认，而不是直接推动用户继续操作。

## 当前能力

- Chrome Manifest V3 插件。
- 普通网页右下角显示“开始语音指导”按钮。
- 点击开始后进入持续语音会话。
- AI 播报时支持手动点击“打断”。
- 回答语言跟随用户输入：
  - 中文提问 -> 中文回答
  - 英文提问 -> 英文回答
- 支持 DeepSeek OpenAI-compatible Chat Completions API。
- 基于 DOM 的页面快照：
  - 页面标题和 URL
  - 可见标题
  - 页面主要可见文本样本
  - 全页面可见的按钮、链接、输入框、文本框、下拉框、可编辑区域
  - label、placeholder、ARIA label、附近文本、元素位置
- 鼠标上下文：
  - 最近鼠标坐标
  - 鼠标下方元素
  - 鼠标附近的交互元素
- 选中文本上下文：
  - 当前鼠标拖动选中的文本
  - 选中文本附近的页面上下文
- 页面高亮：
  - 高亮模型选中的 DOM 元素
  - 如果目标元素不在当前屏幕内，会先自动滚动到该元素附近再高亮
  - 点击页面、滚动页面、超时或结束会话时自动消失
- 敏感字段脱敏：
  - password、token、secret、API key、card 等字段不会发送真实值
- 单一语音会话：
  - 在新标签页启动语音指导时，会自动关闭旧标签页的语音指导
- 可拖动悬浮按钮：
  - 位置会保存在 Chrome 本地存储中

## 工作原理

整体流程：

```text
用户说话
  -> 浏览器语音识别转成文本
  -> content script 构建压缩页面快照
  -> background script 调用 DeepSeek
  -> 模型返回 JSON：answer、targetElementIds、riskLevel、nextStep
  -> 插件语音播报 answer，并高亮相关元素
```

插件不会把完整原始 HTML 直接发给模型，而是发送压缩后的页面语义快照。

## 模型输入和输出

模型会收到：

- 用户问题
- 当前页面快照
- 鼠标上下文
- 最近几轮对话

模型必须返回：

```json
{
  "answer": "这个按钮会为当前项目创建一个新的 API key。",
  "targetElementIds": [3],
  "riskLevel": "caution",
  "nextStep": ""
}
```

字段说明：

- `answer`：要播报给用户的回答
- `targetElementIds`：需要高亮的页面元素 ID
- `riskLevel`：`normal`、`caution` 或 `dangerous`
- `nextStep`：可选；只有用户明确问“怎么做、点哪里、下一步”时才填写

## 下载与安装教程

当前项目是一个未上架 Chrome Web Store 的 MVP，需要通过“开发者模式”手动安装。

### 1. 下载项目

如果你从 GitHub 获取项目，可以使用任意一种方式：

**方式 A：下载 ZIP**

1. 打开项目 GitHub 页面。
2. 点击绿色的 `Code` 按钮。
3. 点击 `Download ZIP`。
4. 解压 ZIP 文件到本地文件夹。

**方式 B：使用 Git 克隆**

```bash
git clone <your-repo-url>
```

然后进入项目目录。

### 2. 打开 Chrome 扩展管理页

1. 打开 Chrome 或 Edge。
2. 访问 `chrome://extensions/`。
3. 开启“开发者模式”。

### 3. 加载插件

1. 点击“加载未打包的扩展程序”。
2. 选择本项目目录。
3. 看到 `Voice Web Guide` 出现在扩展列表中，说明安装成功。

注意：需要选择包含 `manifest.json` 的目录，不要选到上一级目录，也不要直接选择 ZIP 压缩包。

## 配置

当前 MVP 需要用户自己申请并填写 DeepSeek API Key。插件不会自带模型额度，也不会替用户提供 API Key。

### 1. 申请 DeepSeek API Key

1. 打开 DeepSeek 平台。
2. 注册或登录账号。
3. 进入 API Keys 页面。
4. 创建一个新的 API Key。
5. 确认账户有可用余额。

### 2. 在插件中填写配置

1. 点击浏览器右上角扩展图标。
2. 找到并点击 `Voice Web Guide`。
3. 填入 DeepSeek API Key。
4. 默认 endpoint：

```text
https://api.deepseek.com/chat/completions
```

5. 默认模型：

```text
deepseek-chat
```

6. 点击保存。

如果没有 DeepSeek API Key，需要先到 DeepSeek 平台注册账号、创建 API Key，并确认账户有可用余额。

## 使用教程

### 1. 打开普通网页

插件只能在普通 `http` 或 `https` 网页中使用。

可以先用这些页面测试：

- `https://example.com`
- GitHub 页面
- 英文文档页面
- SaaS 控制台页面

不要在这些页面测试：

- `chrome://extensions/`
- Chrome Web Store
- 新标签页
- 浏览器设置页
- 内置 PDF 页面

### 2. 刷新网页

安装或重新加载插件后，已经打开的网页需要刷新一次，插件才会注入页面。

刷新后，页面右下角会出现“开始语音指导”按钮。

### 3. 开始语音指导

1. 点击右下角“开始语音指导”。
2. 浏览器如果请求麦克风权限，请选择允许。
3. 看到状态变成“正在听...”后，就可以自然提问。

例如：

   - “这个页面是干什么的？”
   - “我应该点哪里申请 API key？”
   - “我鼠标指的这个字段是什么意思？”
   - “Callback URL 是什么意思？”
   - “What does this button do?”

插件会语音回答，并高亮相关页面元素。

### 4. 使用鼠标指向提问

如果你不知道某个按钮或字段叫什么：

1. 把鼠标移动到它上面。
2. 直接问：“这个是什么意思？”或“这里应该填什么？”

插件会把鼠标附近的页面上下文传给模型。

### 5. 使用选中文本提问

如果你想解释一段文字：

1. 用鼠标拖动选中网页上的文本。
2. 问：“这段是什么意思？”或“帮我解释一下这段英文。”

插件会把选中的文本传给模型。

### 6. 停止播报或结束会话

- 如果 AI 正在播报，但你不想继续听，点击“打断”。
- 如果任务完成，点击“结束”。
- 点击“结束”后，麦克风会停止监听，悬浮状态框会消失。

### 7. 长页面自动定位

如果模型认为目标按钮或字段在页面下方，插件会自动滚动到目标元素附近，并进行高亮。

如果返回的多个目标里已经有元素在当前屏幕内，插件会优先高亮当前屏幕内的元素，不会立刻滚动页面。

## 更新插件

如果你修改了代码，或从 GitHub 拉取了新版本：

1. 打开 `chrome://extensions/`。
2. 找到 `Voice Web Guide`。
3. 点击“重新加载”按钮。
4. 回到测试网页并刷新页面。

如果只是修改了 README，不需要重新加载插件。

## 卸载插件

1. 打开 `chrome://extensions/`。
2. 找到 `Voice Web Guide`。
3. 点击“移除”。

## 常见问题

### 页面上没有出现“开始语音指导”按钮

可能原因：

- 当前页面不是普通网页，例如 `chrome://` 页面或 Chrome Web Store。
- 安装插件后没有刷新网页。
- 插件没有启用。
- 当前网页限制了扩展脚本注入。

### 点击开始后不能语音输入

可能原因：

- 没有允许麦克风权限。
- 当前浏览器不支持 `SpeechRecognition`。
- 系统麦克风被其他应用占用。
- 当前页面不是可注入的普通网页。

### 模型没有回答或提示请求失败

可能原因：

- 没有填写 DeepSeek API Key。
- API Key 填错。
- DeepSeek 账户没有余额。
- 网络无法访问 DeepSeek API。
- endpoint 或 model 配置错误。

### 高亮位置不准确

可能原因：

- 页面上有多个相似按钮。
- 页面内容在跨域 iframe 中，插件无法读取。
- 页面是 canvas、图片或复杂视觉 UI，DOM 信息不足。
- 模型选错了元素 ID。

### 英文术语被识别或理解不准

当前语音识别依赖浏览器原生能力。中文句子里夹杂英文术语时，浏览器 ASR 可能识别不稳定。

模型侧已经要求保留 API key、Callback URL、Client Secret、Webhook、OAuth 等术语原文，但输入识别质量仍取决于浏览器。

## 局限性

这是一个 MVP，能力边界很明确：

- 只支持普通 `http` 和 `https` 网页。
- 不支持 `chrome://` 页面、Chrome Web Store、新标签页、内置 PDF 页面。
- 主要依赖 DOM 理解页面，不是真正看屏幕截图。
- 对 canvas、图片型页面、视频、游戏、复杂图表、强视觉 UI 支持有限。
- 跨域 iframe 内部内容可能无法读取。
- 浏览器原生语音识别效果会受系统、浏览器、语言、麦克风和网络影响。
- 浏览器内置 TTS 的真人感有限，不如商业神经语音服务。
- 当页面上有很多相似按钮时，高亮可能不准确。
- 模型仍可能误解页面，涉及付款、删除、密钥、权限等操作时，用户必须自己确认。

## 隐私说明

这个 MVP 只在用户提问时，将压缩后的页面快照和用户问题发送到配置的 DeepSeek-compatible endpoint。

插件不会主动保存网页内容。API Key 存储在 Chrome extension storage 中。password、token、secret、API key 等敏感字段会在本地脱敏后再进入页面快照。

当前版本需要用户自行配置 DeepSeek API Key，因此相关模型调用费用由用户自己的 DeepSeek 账户承担。
